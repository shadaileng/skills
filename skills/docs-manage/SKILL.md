## 目录结构

```
docs/
├── README.md                        # 文档索引与执行总览（必选）
├── _template.md                     # 文档创建模板（必选）
├── index.md                         # VitePress 首页
├── .vitepress/                      # VitePress 配置与主题
│   ├── config.ts                    # 站点配置（nav/sidebar）
│   └── theme/                       # 自定义主题
├── plans/                           # 方案类（待执行 → 已完成）
│   ├── 01-xxx.md
│   └── 02-xxx.md
├── architecture/                    # 架构类（持续维护）
│   └── 01-xxx.md
├── references/                      # 参考类（持续维护）
│   ├── 01-xxx.md
│   └── 02-xxx.md
├── guides/                          # 指南/手册类（持续维护）
│   ├── 01-xxx.md
│   └── 02-xxx.md
└── reference/                       # 参考代码（项目特有，.gitignore，不纳入版本管理）
```

### 子目录说明

| 子目录 | 类型 | 生命周期 | 用途 |
|:-------|:----|:---------|------|
| `plans/` | 方案 | 待执行 → 已完成 | 功能实施前的设计方案，含技术调研和实现步骤 |
| `architecture/` | 架构 | 持续维护 | 系统整体的技术架构说明 |
| `references/` | 参考 | 随代码更新 | API、数据字典等开发者参考信息 |
| `guides/` | 指南/手册 | 持续维护 | 开发指南、用户手册等流程指引 |

### 文档类型

| 类型 | 说明 | 生命周期 |
|:----|------|---------|
| **方案** | 功能实施前的设计方案，含技术调研和实现步骤 | 实施完成后 → 🏁 已完成 |
| **架构** | 系统整体的技术架构说明 | 持续维护 |
| **参考** | API、数据字典等开发者参考信息 | 随代码更新 |
| **指南** | 开发/使用流程指引 | 持续维护 |
| **手册** | 面向最终用户的功能说明 | 随功能更新 |
| **模板** | 创建新文档的标准模板 | 按需完善 |

## 创建规则

### 编号

- 每个子目录内独立编号，从 `01` 开始递增
- 跨子目录不要求全局唯一
- 新增文档在当前子目录最大编号后顺延

```
plans/          01-xxx.md  02-xxx.md  03-xxx.md  ...
architecture/   01-xxx.md  02-xxx.md  ...
references/     01-xxx.md  02-xxx.md  ...
guides/         01-xxx.md  02-xxx.md  ...
```

### 索引文件同步

每次增删文档后必须同步更新以下两个索引文件：

| 文件 | 路径 | 需更新内容 |
|:-----|:-----|:-----------|
| VitePress 首页 | `docs/index.md` | 「文档一览」表格行 + 「版本关联」日志 + 「执行进度」状态 |
| 侧边栏配置 | `docs/.vitepress/config.ts` | 对应子目录的 `sidebar` 数组项链接 |

**原则**：增删文档不提交，除非索引文件已同步。可拆分为两个 commit（先加文档、再更新索引），但最终不能出现文档存在但索引缺失的状态。

**双向校验**：修改 `index.md` 后必须同步检查 `config.ts` 侧边栏，反之亦然。两条索引应始终包含相同的文档集。

### 独立编号规则

**核心原则**：每个子目录的编号独立管理，互不影响。

#### 何时需要重新编号

| 场景 | 操作 |
|------|------|
| 文档从全局编号改为子目录独立编号 | 重命名文件 + 更新侧边栏 + 更新 README.md |
| 子目录内编号不连续（如 01, 03, 05） | 重命名文件使其连续（01, 02, 03） |
| 文档移动到其他子目录 | 重新编号为目标子目录的下一个序号 |
| 删除文档导致编号空缺 | 可选：重命名后续文档使其连续 |

#### 重新编号操作流程

```bash
# 1. 进入目标子目录
cd docs/plans/

# 2. 批量重命名（示例：将 09-31 重编号为 01-23）
mv 09-xxx.md 01-xxx.md
mv 10-xxx.md 02-xxx.md
# ... 依此类推

# 3. 更新侧边栏配置
# 编辑 docs/.vitepress/config.ts 中的 sidebar 配置

# 4. 更新 README.md 文档索引
# 编辑 docs/README.md 中的文档一览表
```

#### 注意事项

1. **重命名后必须同步更新**：
   - `.vitepress/config.ts` 侧边栏链接
   - `README.md` 文档索引表
   - 文档内的交叉引用链接（如有）

2. **编号格式**：
   - 始终使用 2 位数字（01-99）
   - 不允许跳号（如 01, 03, 05 应改为 01, 02, 03）

3. **特殊情况**：
   - 如果文档数量超过 99 篇，可扩展为 3 位数字（001-999）
   - 跨子目录编号不要求全局唯一（如 plans/01 和 architecture/01 可以同时存在）

### 命名

```
{序号}-{中文标题}.md
```

- 序号：2 位数字（01-99）
- 中文标题：简洁明了，不超过 15 字
- 文件名中不允许空格

### 模板

创建新文档复制 `_template.md` 到对应子目录。

## 文档格式

### 文档头信息

每个文档开头必须包含引用块格式的本页信息表格：

```
> **本页信息**
>
> | 项目 | 内容 |
> |------|------|
> | 文档编号 | {序号} |
> | 文档版本 | v1.0.0 |
> | 文档状态 | 🏁 已完成 / 🚧 进行中 / 📋 待执行 |
> | 最后更新 | {YYYY-MM-DD} |
> | 对应内容 | {简要描述} |
>
> **变更历史**
>
> | 日期 | 版本 | 说明 |
> |------|:----:|------|
> | {YYYY-MM-DD} | v1.0.0 | 初版 |
>
> **关联文档**：{关联文档链接}
```

### 版本号

```
v{major}.{minor}.{patch}
```

| 变更类型 | 版本影响 | 示例 |
|:---------|:---------|:-----|
| 重大内容重构或范围变更 | MAJOR +1 | v1.0.0 → v2.0.0 |
| 功能增补或章节调整 | MINOR +1 | v1.0.0 → v1.1.0 |
| 勘误、格式调整 | PATCH +1 | v1.0.0 → v1.0.1 |

### 状态标记

| 标记 | 含义 | 适用文档类型 |
|:----:|:-----|:------------|
| 🏁 已完成 | 实施完毕，内容冻结 | 方案（终态） |
| 🚧 进行中 | 正在实施或维护 | 所有类型 |
| 📋 待执行 | 已规划但未开始 | 方案（初始态） |
| ⏳ 已归档 | 过时或废弃 | 所有类型 |

## 变更管理

### 变更历史

每次文档内容更新后，在「变更历史」表格中追加一行：

```
| {YYYY-MM-DD} | {新版本号} | {变更说明} |
```

- 按时间倒序排列（最新在上）
- 变更说明简洁明确
- 每次写入新行，不修改历史行

### 版本关联

当文档变更涉及应用功能发布时，在 `README.md` 的「版本关联」表中同步记录。

## 交叉引用

### 规则

- 相关文档之间必须添加交叉引用链接
- 链接格式：`[文档标题](../子目录/文件.md)`
- 同一子目录内：`[文档标题](./文件.md)`
- 引用位置：文档头信息「关联文档」行 + 正文相关段落末尾

### 示例

```
> **关联文档**：[01-API 参考.md](../references/01-API%20参考.md)（接口定义）

详见 [02-数据字典.md](../references/02-数据字典.md)（字段说明）
```

---

## README.md 要求

docs/ 目录的 README.md 必须包含以下章节：

| 章节 | 说明 |
|:-----|------|
| 目录结构 | ASCII 树形目录，标注子目录说明 |
| 文档一览 | 表格列出所有文档（名称、版本、类型、位置、说明、状态） |
| 版本关联 | 应用版本与文档变更对照表（可选，有版本发布时更新） |
| 执行进度 | 方案类文档的实施步骤跟踪表 |
| 约定 | 编号、状态标记、版本号等规则的速查 |

## 生命周期

```
📋 待执行 → 🚧 进行中 → 🏁 已完成 → （持续维护/归档）
```

- **方案类文档**：初始状态 📋 → 实施中 🚧 → 完成后 🏁（终态）
- **架构/参考/指南/手册类文档**：持续维护，不设终态
- **过时废弃的文档**：在信息表中标注 ⏳ 已归档，不移除

## 操作流程

### 创建新文档

1. 确定文档类型，选择对应子目录
2. 复制 `_template.md` 到目标子目录
3. 按当前子目录最大编号 +1 确定序号
4. 填写「本页信息」表格
5. 编写正文内容
6. 添加「关联文档」交叉引用
7. 更新 `docs/index.md`「文档一览」表格（新增行）
8. 更新 `docs/.vitepress/config.ts` 侧边栏配置（新增链接）
9. 如有版本发布，更新「版本关联」表

> **索引同步原则**：文档可以分步提交，但最终 commit 不能出现文档存在但 `index.md` 或 `config.ts` 未同步的状态。

### 更新现有文档

1. 更新正文内容
2. 「变更历史」中追加一行（新版本号 + 变更说明）
3. 更新头部版本号
4. 更新 `docs/index.md` 中的状态/版本（如有变化）
5. 如文档被移除，同步更新 `docs/index.md`「文档一览」移除对应行 + 更新 `docs/.vitepress/config.ts` 移除对应链接

### 实施方案文档

1. 创建时状态为 📋 待执行
2. 开始实施后更新为 🚧 进行中
3. 实施完毕后更新为 🏁 已完成
4. 实施期间每次提交对应功能的代码后，同步更新文档的「执行进度」
